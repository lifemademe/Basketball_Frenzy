# 11 → 12

## 1. `BaseGameLoopOptions.gameContextConfig` removed

Lift `defaultGameModeClass`, `initialWorldPath`, and any other previously nested fields to the top level of `BaseGameLoopOptions`.

```ts
// Before
new MyGame(container, {
  ...options,
  gameContextConfig: { ...options?.gameContextConfig, defaultGameModeClass: MyGameMode },
});

// After
new MyGame(container, {
  ...options,
  defaultGameModeClass: MyGameMode,
});
```

## 2. `BaseGameLoop.createLoadingScreen()` override removed

Delete any subclass override that only returns `new ENGINE.DefaultLoadingScreen()`. Keep the override only when it returns a custom loading screen.

```ts
// Delete from a BaseGameLoop subclass
protected override createLoadingScreen(): ENGINE.ILoadingScreen | null {
  return new ENGINE.DefaultLoadingScreen();
}
```

## 3. `World` physics and navigation accessors removed

`World.getPhysicsEngine()`, `World.getNavigationServer()`, `World.createPhysicsEngine()`, `World.createNavigationServer()`, and the `useManifold` world option are gone. Read subsystems through `gameLoop`.

```ts
// From a World reference
const phys = world.gameLoop?.physicsEngine ?? null;
const nav  = world.gameLoop?.navigationServer ?? null;

// From inside a BaseGameLoop subclass
const phys = this.physicsEngine;
```

Manifold is always available via `gameLoop.manifold`.

## 4. `BaseGameLoop` getters replaced with properties

Drop the parentheses and rename the call site:

- `getRootScene()` → `rootScene`
- `getManifold()` → `manifold`
- `getPhysicsEngine()` → `physicsEngine`
- `getNavigationServer()` → `navigationServer`
- `getPostProcessManager()` → `postProcessManager`
- `getActiveScenePath()` → `activeScenePath` (read)
- `setActiveScenePath(path)` → `activeScenePath = path` (write)

## 5. `Renderer.render()` removed; rendering owned by `BaseGameLoop.tick()`

Do not call `renderer.render(world)` or `world.renderScene()` from a custom tick. Call `super.tick(t)` and rendering happens automatically.

```ts
// Before
protected override tick(t: ENGINE.TickTime): void {
  this.world.tick(t);
  this.renderer.render(this.world);
}

// After
protected override tick(t: ENGINE.TickTime): void {
  super.tick(t);
  // custom logic
}
```

## 6. Scene environment API replaced by `ISceneEnvironmentSource`

`World.getSceneEnvironment()`, `World.setSceneEnvironment()`, `SceneEnvironmentComponent.enableTexture()`, and `SceneEnvironmentComponent.disableTexture()` are removed. The engine's root scene is now an `ENGINE.GenesysScene` that aggregates environment sources per frame.

For any custom component that wrote to `THREE.Scene.background` or `THREE.Scene.environment` directly:

1. Implement `ENGINE.ISceneEnvironmentSource`.
2. Call `genesysScene.addSceneEnvironment(this)` on activation and `removeSceneEnvironment(this)` on deactivation.

Reference implementations: `SkyComponent` and `SceneEnvironmentComponent` in `.engine/src/components/visual/`.

## 7. Legacy module aliases removed

Use `@gnsx/genesys.js` only. The engine no longer resolves:

- `'genesys.js'`
- `'@directivegames/genesys.js'`

Update any dynamic `import(...)` or script that still references the old names.

## 8. Deprecated in 12 (still works)

- `ModelMeshComponent.getModel()` — use the component instance directly as the scene-graph root.
